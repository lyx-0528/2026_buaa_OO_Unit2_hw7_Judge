# 2026_buaa_OO_Unit2_hw7_Judge
## 🚀 自动化评测机脚本 (Batch Judge Tool)

这是一个专为 buaa OO 课程设计的轻量级评测工具，支持自动生成数据、重复性评测以及错误数据自动归档功能。

---

## 📂 项目结构说明

请确保你的项目根目录下包含以下文件：
- `gen_data.bat`：核心评测脚本
- `generator.py`：数据生成器（产生输入数据）
- `checker.py`：结果校验器（对比或检查输出是否正确）
- **`src.jar`**：**你的待测 Java 程序（详见下方封装说明）**
- `error/`：(自动生成) 用于存放评测失败的记录

---

## 🚀 使用指南

### 1. 封装 Java 项目 (重要)
请将你的 Java 项目编译并打包为 **`src.jar`**。

> [!CAUTION]
> **封装要求：**
> - **命名限制**：JAR 包文件名必须严格命名为 `src.jar`。
> - **数据投喂**：程序内部必须包含**数据投喂器逻辑**，即程序应能够直接通过标准输入 (`System.in`) 读取数据,你可以在讨论区得到投喂器。
> - **放置位置**：将打包好的 `src.jar` 直接放在与 `gen_data.bat` 同级的根目录下。

### 2. 开始评测
双击运行 `gen_data.bat`，根据提示选择模式：

- **模式 [1] 正常评测**：
  - 输入数据组数（需要运行多少组不同的测试用例）。
  - 输入重复次数（每组数据重复运行几次，用于检测随机性错误）。
- **模式 [2] 错误重测**：
  - 脚本会列出 `error/` 文件夹下所有已记录的错误文件夹。
  - 输入编号即可针对该特定错误用例进行多次复测，方便调试。

### 3. 查看错误归档
当程序出现 **WA** (答案错误)、**TLE** (超时) 或 **RE** (运行崩溃) 时，脚本会自动创建如下结构的文件夹：
`error/case_月日_时分秒_错误类型/`

每个文件夹内包含：
- `input.txt`：触发错误的输入数据。
- `output.txt`：程序运行时的错误输出。
- `info.txt`：记录了具体的错误类型和运行标签。

---


PS：你可能需要很多次运行才能找到所有潜在的错误，**请耐心调试和优化你的代码**。评测工具的设计初衷是帮助你发现并修复代码中的问题，而不是一次性通过所有测试。
